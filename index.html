Project Idea

Overview

The proposed project is a control system to adjust the weight of mushroom bags as they are being filled. The control system will be attached to a substrate mixer to automate the opening and closing of a chute as the system receives weight inputs. This will adjust the substrate output and keep bag weights within an acceptable range. The system will also need to account for a delay between the open or close action and the substrate being extruded to the bags.

Motivation

Mushroom yield is significantly affected by the size of the substrate block that is produced. Rashid et al. (2007) suggest that oversized mushroom blocks will limit yields and undersized blocks do not make efficient use of bagging costs. I operate a mushroom farm where we produce 1000 bags of mushroom substrate each week. Our current bagging system requires a person to be available to adjust the substrate flow from a chute every few minutes to keep the bag weight within an optimal range. This means that they must leave the station they are working at, walk to the other end of the shed and adjust the flow. The substrate takes over a minute to reach the bagging machine, so adjustments take a long time to be observed. This means a frequent ‘pendulum effect’ with weights being adjusted multiple times in one direction before realising the adjustment has gone too far and needs to be partially reversed. Automating this adjustment would mean a significant time saving in walking time, and time spent manually adjusting over or underweight bags.

Description

This project would incorporate a software control system to take input from a user in the form of a target weight at the start of operation. It would also regularly take weight data inputs, compare the input weight to the target weight, and send signals to an adjustment mechanism to open or close the chute accordingly, and change the substrate output of the mixer.

A simple user interface will handle the target weight input, provide button options for manually incrementing an open or close action. It will also have a simple start/stop so adjustment can be isolated if needed. The regular weight inputs will come from a load cell onto which mushroom bags will be placed as they are removed from the bagging machine. For every weight input the system will compare to the target weight and send adjustment signals. Larger deviations from the target weight will require larger adjustments, and if the mushroom bags are close enough to the target weight, no adjustment will be made, as small variations in flow could mean constant adjustment signals and unnecessary wear on the adjustment mechanism. After the appropriate adjustment is calculated it will be sent the adjustment mechanism. The adjustment mechanism will be a reversible gear motor attached to a screw drive, to force the chute open or closed as the motor rotates. The adjusted substrate amount will travel through the system and be bagged and placed onto the load cell providing the input for the next adjustment.

Tools & Technologies

The control system will be centred around a Raspberry Pi running a version of ROS (Robot Operating System). Within ROS all software will be written in python (Thank you Intro to Programming) This Rpi will be directly interfaced with over wifi by way of an android phone using ROS android interface tools. This is where target weight will be input, and other interface buttons will reside. The weight inputs will come from a 5kg load cell, and a weight sensor to convert this input to a readable form for the Raspberry Pi. The adjustment mechanism will be a small 12 volt gear motor with an ACME screw and hub nut attached to the chute, and will be connected to a motor controller to allow it to be controlled directly by the Raspberry Pi.

Skills required

Knowledge of ROS and Python on the Raspberry Pi will be essential. I am currently studying Intro to programming where we are working with Python, and ROS is an open source project with extensive modules and documentation readily available (ROS.rog). All hardware is readily available online and I have all the necessary components already, from other project

Outcome

If the project is successful there will be no need for manual adjustment of mushroom chutes. It will no longer be necessary to stop other jobs to make adjustments, and a lot of time will be saved in adjusting oversized or undersized bags. I estimate this will save us 1 hour per bagging run (approximately 3 hours per week). Consistent bag size will lead to consistent yield, and a more productive farm!
